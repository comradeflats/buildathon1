# buildathon.LIVE Platform Ideas

## Vision
A platform specifically focused on **IN-PERSON** live hackathon events, serving as a portal and focal point for organizers worldwide to host their own "buildathons".

## Core Pillars

### 1. The "Live" Portal (Event Discovery)
- **Global Event Directory:** Landing page to discover upcoming in-person buildathons (map view/city search).
- **Vanity Event URLs:** Clean, shareable URLs (e.g., `buildathon.live/e/sf-ai-spring`).
- **Live Event Status:** Real-time badges ("Registration Open", "Hacking", "Judging", "Results Revealed").

### 2. Multi-Tenant Organizer Tools
- **Organizer Workspaces:** Self-serve event creation and management for independent organizers.
- **"Projector Mode":** Dedicated full-screen UI for venues featuring:
    - "Keyboards Down" countdown timer.
    - Auto-scrolling project gallery.
    - Dynamic, animated leaderboard for closing ceremonies.

### 3. In-Person Logistics
- **QR Code Workflows:** Quick scanning for registration, voting, and event info.
- **Judge Delegation:** Temporary PIN-protected links for guest judges with custom voting multipliers.
- **Table Assignments:** Mapping teams to physical locations in the room so judges can find them.

### 4. Hacker Profiles
- **The "Buildathon Resume":** A trophy case of attended events, submissions, and wins tied to GitHub identity.
- **Community Building:** Connect with other builders met during physical events.
